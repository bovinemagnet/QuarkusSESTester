high-reactivity HTMX dashboard for the QuarkusSESTester daemon mode. 

The design focuses on a "Kinetic Terminal" aesthetic—clean, dark, and highly functional for developers.

Here's what I've created:

Dashboard Overview: A real-time hub showing recent email ingress, system health metrics, and active daemon connections.
Send Email Interface: A functional screen to manually dispatch test emails, including fields for metadata, payload (HTML/Plain), and attachments.

Configuration & Logs: A live log streaming view paired with essential daemon settings, allowing you to manage the mock environment without leaving the browser.

Template Gallery: This allows you to manage, preview, and test saved email templates efficiently. You can see the payload size, last usage, and quickly fire off a test email from here.
Node Network: Designed for multi-node setups, this view provides a deep dive into the health of your daemon cluster, showing latency, load, and individual node status in a high-density "terminal" layout.
